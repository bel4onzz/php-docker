Brief code review

Complex Logic in ageByOneDay Method:

The ageByOneDay method has complex nested conditional statements that make it difficult to read and maintain.

Duplicated Code:

There are several blocks of code that are repeated for different item types, especially regarding the updates to value and sellIn. This violates the DRY (Don't Repeat Yourself) principle.

Missing Type Declarations:

The properties $name, $value, and $sellIn lack explicit types.

Method Naming Consistency:

The name() and value() methods are ambiguous as they could imply an action rather than a getter.

Encapsulation Violation via Direct Property Exposure:

Exposing the properties through methods like name() and value() without control may violate encapsulation principles.

Violation of the Single Responsibility Principle:

The ageByOneDay method is responsible for multiple actions: calculating the new value based on the item type, modifying the sellIn value, and handling different conditions for various items. This can lead to difficulties in testing and maintaining the code.
